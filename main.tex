\section{Introduction}  
Notre groupe de travail est composé de trois étudiantes : Dalia El Fhal, Hajar Ajouirja et Eris Miftari.

Ce projet a été réalisé dans le cadre du cours de gestion de projet de notre deuxième année de Licence Informatique à l'Université Lumière Lyon 2.

L'objectif de ce mini-projet est de mettre en pratique l'utilisation des outils de gestion de version, notamment Git et GitHub, dans un contexte de travail collaboratif.

Nous avons choisi de réaliser un projet LaTeX afin de rédiger un rapport structuré et de pouvoir suivre l'évolution du travail à l'aide d'un dépôt GitHub.

Le dépôt du projet est hébergé sur GitHub, ce qui permet de gérer les différentes versions du projet et de faciliter la collaboration entre les membres du groupe.

Les tâches ont été réparties entre les membres du groupe afin de travailler de manière organisée et efficace.


\section{Description du projet}
Ce projet a été réalisé dans le cadre du module de gestion de projet de la deuxième année de Licence Informatique à l'Université Lumière Lyon 2.

L'objectif de cette mise en situation est de découvrir l'utilisation des outils de gestion de version, notamment Git et GitHub, dans un travail collaboratif.

Nous avons choisi de réaliser un rapport en LaTeX afin de présenter notre travail. Le projet est hébergé sur GitHub, ce qui permet de suivre l'historique des modifications et de gérer les contributions de chaque membre du groupe.

Chaque membre du groupe travaille sur une branche spécifique du dépôt. Les modifications sont ensuite intégrées dans la branche principale à l'aide de pull requests.

Cette organisation nous a permis de comprendre le fonctionnement d'un projet collaboratif et l'importance de la gestion de versions.


\section{Conclusion}
Ce projet nous a permis de mettre en pratique l'utilisation de Git et GitHub
dans un contexte de travail collaboratif. La gestion des branches, des commits
et des pull requests nous a permis de mieux comprendre l'organisation d'un projet
informatique.
